Психология тестирования

Качество ПО - способность программного продукта при заданных условиях удовлетворять установленным или предполагаемым потребностям (как явным или неявным)

ISTQB - степень соответствия системы, компонента или процесса

Качественное ПО - не содержит ошибок и дефектов, доставлено вовремя и в рамках бюджета, соответствует требованиям  и/или ожиданиям, и является поддерживаемым.
Зачастую может выиграть тот кто сделает хуже, но быстрее и/или дешевле.

Характеристики качества ISO/IEC 25010
Функциональность - определяется способностью ПО решать задачи, которые соотв. зафиксированным и предполагаемым потребностям пользователя, при заданных условиях использования ПО
Производительность - способность ПО обеспечивать требуемый уровень производительности  в соотв. с выделенными ресурсами, временем и др. обозначенными условиями.
(Производительность крайне важна и может быть очень прибыльной. Например +7-8% к производительности может дать 70-80% прибыли)
Удобство использования - возможность легко понимать, изучать и использовать ПО и привлекательность для пользователя
Совместимость - способность системы обмениваться инфой с другими системами или выполнять  свои функции используя совместно аппаратную или программную среду (например IOS)
Надежность - способность ПО выполнять требуемые задачи на протяжении заданного отрезка времени или указанного количества операция
Безопасность - способность системы обеспечивать доступ только к тем данным, которые разрешены для пользователя или других систем в соответствии с их правами.
Сопровождаемость - легкость, с которой ПО может анализироваться, тестироваться, изменяться, для улучшения, для внедрения новых функций
Переносимость - характеризует ПО с точки зрения легкости его переноса из одного окружения (software/hardware) в другое (адаптивность, устанавливаемость, заменяемость).

Что такое тестирование?
Процесс исследования испытания продукта с целью проверки соответствия между реальным поведением программы и её ожидаемым проведением на конечном наборе тестов, выбранных определенным образом.
Задача тестирования проверить работоспособность, а уже после найти варианты, при которых ПО неработоспособно.
Виды тестирования
Статическое - тестирование при котором программный код НЕ будет выполняться. При этом само тестирования может быть как ручным, так и автоматизированным (вычитка исходного кода, проверка требований)
Динамическое - тип тестирования, который предполагает запуск программного кода .
Цели тестирования
Повышение качества ПО
Предотвращение появления дефектов (т.е. еще на этапе разработки)
Предоставление инфы о качестве ПО конечному заказчику
